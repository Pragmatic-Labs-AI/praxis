# Decision digest

The overview. The full, append-only record is the table under
[§11 Decision record](#11-decision-record) below; fully superseded rows move to
[decisions-archive.md](decisions-archive.md) with their D-numbers intact. This
digest names the load-bearing decisions grouped by theme — read it first, then
drop into the table for the detail and rationale. Numbers in **bold** are the
row(s) to look up.

**Foundations — what Praxis is and how it ships**
- **D1** — a TypeScript/Node CLI, `npx`-first, that runs on any machine with no install.
- **D2 / D4 / D6** — imperative entry, declarative truth: commands do the smart thing, but the committed `praxis.yaml` is the source of state, and new methodology reaches a repo only through an explicit, version-pinned `sync`.

**The differentiated core — composition + drift-checking, not the content (D42)**
- **D7 / D10** — merge is text block-splice with managed-block markers; an edit inside a managed block is a *conflict, never a clobber*.
- **D26 / D27** — harness currency is a PR-time CI gate: a deterministic emitted-file drift check plus knowledge-anchor resolution; the LLM judgment runs in the developer's agent, never in CI.

**Non-negotiable scope boundaries**
- **D9 / D12** — Praxis never writes the target's application code, deps, or infra, and never calls an LLM itself; it ships instructions the ambient agent executes.
- **D8** — "memory" always means the agent's memory of the project, never a runtime memory backend for the app.

**One neutral source → native per-tool surfaces**
- **D19 / D28** — permissions and plugin-marketplaces emit per-tool and quarantined, composed into a single `.claude/settings.json` write with conflict-not-clobber; a tool with no model for an artifact emits nothing.
- **D30** — the manifest is multi-stack and Layer 2 packages gate on declared stacks (the stack-composition shape the workspace tier reuses).
- **D47 / D48** — Claude Code and OpenAI Codex are both first-class and kept *peer-native* (neither imports the other); shared project facts stay in parity.

**Keeping the methodology layer current**
- **D29 / D31** — `/praxis-upkeep` is the single front gate sequencing the drift check, the instruction-layer audit, and this wiki's maintenance.
- **D25** — this knowledge wiki is the agent's durable, in-repo memory *of the project* — curated markdown, no backend (consistent with D8/D12).

**Reach & hardening**
- **D49 / D53** — a repo can ship its own project-local packages, and a meta-repo can be a hub-sovereign workspace over N independent git repos cloned beneath it.
- **D54 / D55** — consumer-repo path/symlink containment and full-commit-SHA-pinned plugin sources (the OSS-launch security batch).
- **D56 / D57 / D58** — the OSS-launch public/release boundary: an allowlist-driven public-snapshot builder plus an executed secret/term sweep, a tag-triggered npm trusted-publishing workflow as the sole publish path, and the in-repo org migration (`@pragmatic-labs/praxis`, Node 22 floor, signed-off snapshot root, recorded first-publish bootstrap).
